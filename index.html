Что такое ClaudeClaw

Простыми словами — зачем это нужно и чем отличается от обычных ботов.

Ваш личный ИИ-ассистент прямо в Telegram

ClaudeClaw — это Telegram-бот, за которым стоит полноценный Claude (тот же, что на claude.ai). Вы пишете ему в Telegram — он отвечает как настоящий ассистент с памятью, голосом и навыками.

В отличие от обычных «клавиатурных» ботов на API, ClaudeClaw работает через Claude Code — бот может искать в интернете, работать с файлами, выполнять код и помнить ваши предыдущие разговоры. Это безопаснее и мощнее, потому что работает напрямую через инфраструктуру Anthropic.

В чём отличие от OpenClaw?

OpenClaw — проект, набравший 100 000 звёзд на GitHub. Делал похожую штуку, но у него оказались серьёзные проблемы: он крал ваш токен авторизации Claude и прогонял через чужой код. Anthropic это запретила — проект по сути забанен.

Подготовка: всё, что нужно заранее

~20–30 мин

Перед установкой подготовьте 5 вещей. Делайте по порядку.

0.1 Создаём сервер на DigitalOcean

DigitalOcean — облачный хостинг, где будет жить ваш бот. Работает 24/7, даже когда компьютер выключен.

  • Зайдите на digitalocean.com и зарегистрируйтесь
  • Нажмите Create → Droplets
  • Выберите Ubuntu 24.04
  • Тариф: $6/мес (1GB) для теста или $12/мес (2GB) для стабильной работы
  • Регион: выбирайте ближе к вашему городу. Например, для Москвы и европейской части России — Амстердам
  • Задайте пароль для root (запишите!)
  • Create Droplet → запишите IP-адрес
0.2 Создаём Telegram-бота

Бот создаётся через @BotFather — специального бота Telegram.

  • Откройте @BotFather в Telegram
  • Напишите /newbot
  • Придумайте имя и username
  • Сохраните токен (строка вида 8258092538:AAGs...)
0.3 Узнаём Chat ID

Chat ID — ваш уникальный номер. Бот будет отвечать только вам.

  • Найдите @userinfobot
  • Напишите /start
  • Сохраните ID (число вида 12345678)
0.4 Groq API ключ (для голосовых)

Groq — полностью бесплатный сервис для распознавания голосовых сообщений. Щедрые лимиты, платить не нужно.

  • Зарегистрируйтесь на groq.com (бесплатно, можно через Google)
  • После регистрации зайдите в консоль → API Keys
  • Нажмите Create API Key
  • Сохраните ключ (gsk_...)
0.5 Аккаунт Claude с подпиской Pro или Max

Для работы бота нужен аккаунт на claude.ai с активной подпиской Pro или Max.

  • Если есть аккаунт на claude.ai — отлично
  • Создать и оплатить аккаунт Claude с русской картой можно по этой ссылке
✅ Чек-лист

У вас должны быть: IP сервера, пароль root, токен бота, Chat ID, Groq ключ, аккаунт Claude.

Результат шага
Все ключи и аккаунты готовы. Приступаем к установке.

Подключение к серверу

~2 мин
Сейчас мы подключимся к серверу через консоль — это утилита для управления сервером прямо в браузере. Не пугайтесь, мы будем просто копировать и вставлять готовые команды — никаких специальных знаний не нужно.

  • Нажмите на ваш дроплет (дроплет — это сервер, который мы создали в Шаге 0)
  • Кнопка «Console» справа вверху
  • Откроется терминал — вы внутри сервера
Результат шага
Вы подключены. Видите root@имя-сервера:~#

Проверка

Установка Claude Code CLI

~5 мин
Claude Code — программа от Anthropic для общения с Claude через командную строку. Через неё бот будет «думать».
Установка
npm install -g @anthropic-ai/claude-code
Запуск и авторизация

После установки нужно запустить Claude Code и пройти авторизацию в своём аккаунте Anthropic.

Запускаем Claude Code
claude
  • При первом запуске Claude Code спросит тему оформления, метод логина и другие настройки — не пугайтесь, просто нажимайте Enter на каждый вопрос, соглашаясь со значениями по умолчанию
  • После этого в консоли появится ссылка для авторизации — она начинается с https://claude.ai/...
  • Скопируйте эту ссылку и откройте в браузере на вашем компьютере или телефоне (на сервере браузера нет)
  • Войдите в свой аккаунт Anthropic (или он уже авторизован)
  • Нажмите «Разрешить» (Allow) — вы даёте Claude Code доступ к вашему аккаунту
  • Вернитесь в консоль сервера — там появится подтверждение, что авторизация прошла
⚠️ Важно

Ссылку нужно открыть именно в браузере на вашем компьютере или телефоне — на сервере браузера нет. Просто скопируйте URL и вставьте в адресную строку.

Проверяем, что всё работает

Отправим тестовое сообщение, чтобы убедиться, что Claude отвечает:

Тестовая команда
claude -p 'скажи привет'
Результат шага
Claude отвечает «Привет!» — авторизация прошла, всё работает.

Генерация проекта ClaudeClaw

~15–20 мин
Claude Code сам напишет весь код бота. Вы просто вставите промпт ClaudeClaw_prompt, а Claude сделает остальное.
Создаём папку и запускаем
mkdir ~/claudeclaw && cd ~/claudeclaw claude
  • Скачайте ClaudeClaw_prompt (кнопка внизу страницы)
  • Откройте и скопируйте весь текст
  • Вставьте в консоль Claude Code
  • Claude прочитает промпт и выдаст длинное описание возможностей бота. В конце он напишет что-то вроде «Any questions before we get into the setup choices?» — не пугайтесь, просто скопируйте и вставьте эту команду:
Вставьте этот ответ
Platform: telegram. Voice: microphone. Memory: full. Features: scheduler, service. Go
💬 Что означают эти настройки

telegram — бот будет работать в Telegram

microphone — распознавание голосовых сообщений через Groq (бесплатно)

full — полная система памяти — бот запоминает вас со временем

scheduler, service — планировщик задач + автозапуск при перезагрузке сервера

  • После вашего ответа Claude начнёт генерировать проект. Подождите 15–20 минут — не закрывайте консоль
⌨️ Подтверждения во время генерации

Пока Claude создаёт файлы проекта, он будет периодически останавливаться и запрашивать подтверждение — например, на создание или изменение файлов. Каждый раз просто нажимайте Enter, чтобы разрешить действие. Это нормальная часть процесса — Claude спрашивает разрешение перед каждой операцией с файлами. Не уходите далеко от консоли.

Результат шага
Проект сгенерирован. Появились файлы: src/, package.json и другие.

Настройка автозапуска (24/7)

~5 мин
Настроим, чтобы бот работал постоянно и перезапускался при сбоях.
Создание сервисаПод root
cat > /etc/systemd/system/claudeclaw.service << 'EOF' [Unit] Description=ClaudeClaw Telegram Bot After=network.target [Service] Type=simple User=claw WorkingDirectory=/home/claw/claudeclaw ExecStart=/usr/bin/node dist/index.js Restart=always RestartSec=5 Environment=PATH=/usr/bin:/usr/local/bin [Install] WantedBy=multi-user.target EOF
Активируем
systemctl daemon-reload systemctl enable claudeclaw systemctl start claudeclaw
Проверяем
systemctl status claudeclaw
Вы должны увидеть
● claudeclaw.service — active (running). Бот работает 24/7!

Добавление скиллов

~5 мин
Скилл — это обычный текстовый файл с расширением .md, в котором написана инструкция для бота. Например, скилл «копирайтер» может содержать правила написания текстов, tone of voice, примеры. Когда вы отправляете боту задачу, он автоматически подключает нужный скилл.
Как добавить скилл через VS Code
  • Подключитесь к серверу через VS Code (Шаг 10)
  • В левой панели раскройте папку .claude → skills
  • Если папки skills нет — создайте: правый клик на .claude → New Folder → skills
  • Правый клик на skills → New File → назовите файл, например copywriter.md
  • Вставьте в файл ваш промпт — инструкцию для бота. Как в Notion: просто текст с правилами, стилем, примерами
  • Сохраните: Cmd+S (Mac) / Ctrl+S (Win)
✅ Перезапуск не нужен

Скиллы подхватываются автоматически при следующем сообщении боту. Просто сохранили файл — и всё работает.

💡 Идеи скиллов

Копирайтер, аналитик конкурентов, SMM-менеджер, email-маркетолог, генератор хуков, ДНК клиента — любая роль, которую вы хотите дать боту. Один скилл = один файл.

Результат шага
У бота появился новый навык. Отправьте ему задачу по теме скилла — он подключит его автоматически.
